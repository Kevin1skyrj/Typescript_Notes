# Lecture 3 - TypeScript Setup & Configuration

## Installing TypeScript

### Project-Level Installation (Recommended)

```bash
npm init -y

npm install -D typescript
```

### Global Installation

```bash
npm install -g typescript
```

> Project-level installation is preferred because different projects may require different TypeScript versions.

---

# TypeScript Compiler (TSC)

After installing TypeScript, the TypeScript Compiler (TSC) becomes available.

### Initialize TypeScript

```bash
npx tsc --init
```

This generates:

```text
tsconfig.json
```

---

# tsconfig.json

The `tsconfig.json` file controls how TypeScript behaves.

Example:

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./dist",
    "strict": true
  }
}
```

---

## Important Compiler Options

### rootDir

Specifies where TypeScript source files are located.

```json
{
  "rootDir": "./src"
}
```

Example:

```text
src/
 └── index.ts
```

---

### outDir

Specifies where compiled JavaScript files will be generated.

```json
{
  "outDir": "./dist"
}
```

Example:

```text
dist/
 └── index.js
```

---

### strict

Enables strict type checking.

```json
{
  "strict": true
}
```

> Recommended for all projects.

---

### target

Specifies the JavaScript version generated by TypeScript.

```json
{
  "target": "ES2020"
}
```

Common values:

```text
ES5
ES6
ES2017
ES2020
ESNext
```

# Compiling TypeScript

```bash
npx tsc
```

This command:

1. Reads tsconfig.json
2. Compiles TypeScript
3. Generates JavaScript inside `dist`

Example:

```text
src/index.ts
        ↓
      npx tsc
        ↓
dist/index.js
```

---

# Generated Files

After compilation, TypeScript may generate:

```text
dist/
├── index.js
├── index.d.ts
└── index.js.map
```

---

## index.js

Compiled JavaScript file.

```js
function greet(person) {
  return `Hello ${person}`;
}
```

---

## index.d.ts

Declaration file.

Contains type definitions.

Example:

```ts
declare function greet(person: string): string;
```

---

## index.js.map

Source map file.

Used for debugging.

Maps JavaScript back to original TypeScript source code.

---

# Running Compiled JavaScript

```bash
node dist/index.js
```

# Package.json Scripts

### Build Script

```json
{
  "scripts": {
    "build": "npx tsc"
  }
}
```

Run:

```bash
npm run build
```

---

### Start Script

```json
{
  "scripts": {
    "start": "node dist/index.js"
  }
}
```

Run:

```bash
npm run start
```

---

# ts-node

`ts-node` allows running TypeScript directly without manually compiling first.

### Installation

```bash
npm install -D ts-node
```

### Run TypeScript

```bash
npx ts-node src/index.ts
```

# Interview Questions

## What is TSC?

TSC (TypeScript Compiler) converts TypeScript into JavaScript.

---

## What does `npx tsc --init` do?

Creates a `tsconfig.json` file.

---

## What is tsconfig.json?

Configuration file for the TypeScript compiler.

---

## What is the difference between rootDir and outDir?

### rootDir

Location of TypeScript source files.

### outDir

Location of generated JavaScript files.

---

## What does strict: true do?

Enables strict type checking and catches more errors during development.

---

## What is ts-node?

A package that allows executing TypeScript directly without manually compiling it.

---

## What is a .d.ts file?

A declaration file that contains type information.

---

## What is a source map (.map)?

A debugging file that maps generated JavaScript back to TypeScript source code.

---
