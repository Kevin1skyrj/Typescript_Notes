# Lecture 2 - TypeScript Compilation Process

## TypeScript Compilation Flow

```text
TypeScript (.ts)
        ↓
      Lexer
        ↓
      Parser
        ↓
      Binder
        ↓
      Checker
        ↓
      Emitter
        ↓
JavaScript (.js)
```

---

## Important Concept

TypeScript never runs directly.

```text
TypeScript → Compilation → JavaScript → Execution
```

Browsers and Node.js execute JavaScript, not TypeScript.

---

# 1. Lexer (Scanner)

The first stage of compilation.

### Responsibility

Converts source code into tokens.

Example:

```ts
function greet(name: string) {
    return name;
}
```

Tokens:

```text
function
greet
(
name
:
string
)
{
return
name
}
```

### Detects Basic Syntax Errors

Examples:

- Missing closing quote
- Missing bracket
- Missing parenthesis
- Invalid keywords

---

# 2. Parser

The parser receives tokens from the lexer.

### Responsibility

Creates an AST (Abstract Syntax Tree).

### AST

AST = Abstract Syntax Tree

A tree representation of source code.

Example:

```ts
function greet(name: string) {
    return name;
}
```

Parser converts this code into a tree structure that TypeScript can understand internally.

### Interview Point

AST stands for Abstract Syntax Tree.

Parser is responsible for generating the AST.

---

# 3. Binder

The Binder is a TypeScript-specific stage.

### Responsibility

Creates relationships between symbols and nodes.

### Main Tasks

#### Symbol Tables

Stores information about:

- Variables
- Functions
- Interfaces
- Types
- Classes

Example:

```ts
let username: string = "Rajat";
```

Binder records:

```text
username → string
```

---

#### Parent Pointers

Allows traversal from child nodes back to parent nodes inside the AST.

---

#### Flow Nodes

Used for control-flow analysis.

Example:

```ts
if (typeof name === "string") {
    return name;
}
```

Binder creates flow information that helps TypeScript understand possible execution paths.

---

# 4. Checker

Most important phase of TypeScript.

### Responsibility

Performs type checking.

### What It Checks

- Type Safety
- Type Compatibility
- Type Inference
- Assignments
- Function Signatures
- Interfaces
- Generics

Example:

```ts
let username: string = "Rajat";

username = 42;
```

Error:

```text
Type 'number' is not assignable to type 'string'
```

### Why TypeScript Exists

The Checker is the primary reason TypeScript exists.

Without the Checker, TypeScript would simply be JavaScript.

---

## Interview Point

The Checker performs static type analysis before execution.

---

# 5. Emitter

Final stage of compilation.

### Responsibility

Generates JavaScript code.

### What It Does

Removes TypeScript-specific syntax.

Example:

### TypeScript

```ts
function greet(name: string): string {
    return name;
}
```

### Generated JavaScript

```js
function greet(name) {
    return name;
}
```

Removed:

```ts
: string
```

because JavaScript does not understand types.

---

## Emitter Also

Generates:

```text
.js
.d.ts
.map
```

Files depending on configuration.

---

# Generated Files

## .js

Compiled JavaScript.

Example:

```text
app.ts
↓
app.js
```

---

## .d.ts

Declaration File

Contains type information for other TypeScript projects.

Example:

```ts
declare function greet(name: string): string;
```

---

## .map

Source Map File

Used for debugging.

Maps generated JavaScript back to original TypeScript code.

---

# Key Takeaways

- TypeScript never executes directly.
- Lexer converts code into tokens.
- Parser creates AST.
- Binder creates symbol tables and flow information.
- Checker performs type checking.
- Emitter generates JavaScript.
- JavaScript is the final executable output.

---

# Interview Questions

## What are the stages of TypeScript compilation?

```text
Lexer
↓
Parser
↓
Binder
↓
Checker
↓
Emitter
```

---

## What is AST?

AST (Abstract Syntax Tree) is a tree representation of source code generated by the parser.

---

## What does the Binder do?

Creates symbol tables, parent pointers, and flow nodes.

---

## What does the Checker do?

Performs static type checking and type validation.

---

## What does the Emitter do?

Converts TypeScript into JavaScript by removing type annotations.

---

## Which component is responsible for Type Checking?

Checker

---

## Does Node.js execute TypeScript directly?

No.

TypeScript is converted into JavaScript first, then executed.

---

## Why is the Checker important?

It catches type-related errors before runtime.
